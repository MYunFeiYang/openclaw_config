# 数据分析结果

## ACPI补丁有效性分析
- **补丁功能**：修改后的`GPRW`方法强制返回固定电源状态值，可解决因合盖检测逻辑错误导致的睡眠唤醒失败
- **潜在风险**：可能影响依赖动态电源管理的设备（如USB外设唤醒功能）
- **建议验证步骤**：
  1. 将补丁编译为`SSDT-GPRW.aml`并放入`EFI/OC/ACPI`目录
  2. 在`config.plist`中启用该补丁并禁用`SSDT-LID-G7.aml`
  3. 设置内核参数`darkwake=1`后重启测试

## 睡眠日志深度解读
- **核心问题**：`EC.LidOpen`唤醒事件表明物理 lid 传感器存在误触发（可能由硬件接触不良或固件bug导致）
- **辅助证据**：系统日志中未发现其他唤醒源（如USB/RTC），排除外部设备干扰
- **长期解决方案**：
  - 升级BIOS至最新版本（当前F.23可能存在传感器驱动问题）
  - 若硬件问题无法解决，可通过`SSDT-LID-G7.aml`自定义lid检测阈值

## 实施优先级建议
1. 立即应用ACPI补丁和内核参数修改（10分钟完成）
2. 收集72小时睡眠测试数据（观察唤醒频率变化）
3. 若问题复现，执行`log show --predicate "process == 'kernel' AND eventMessage CONTAINS 'Lid'" --last 24h`捕获传感器原始数据
