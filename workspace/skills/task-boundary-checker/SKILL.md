# 复杂任务边界校验技能（精准版）

## 🎯 核心定位

**不是拦截器，而是智能导航仪** - 帮助用户把模糊需求变成可执行任务，避免返工和误解。

## ✅ 精准触发条件

### **触发场景（满足任一即生效）**

#### **1. 任务类需求 - 动词+模糊目标**
- 包含：「做/开发/分析/设计/规划/构建/实现/优化」
- **且** 未明确：目标/约束/范围/验收标准
- **示例**：
  - ✅ "帮我开发一个用户管理系统" ❌ 未明确功能范围
  - ✅ "分析一下销售数据" ❌ 未明确分析维度
  - ❌ "Python列表怎么排序" ✅ 明确具体技术点

#### **2. 决策类需求 - 选择+无判断维度**
- 包含：「选哪个/怎么选/是否可行/哪个更好」
- **且** 无：比较维度/评估标准/约束条件
- **示例**：
  - ✅ "选哪个框架好？" ❌ 无业务场景
  - ✅ "这个方案是否可行？" ❌ 无可行性标准
  - ❌ "React和Vue哪个性能好" ✅ 有具体比较维度

#### **3. 模糊类需求 - 笼统指令**
- 包含：「帮我处理XX」「优化XX」「完善XX」
- **且** 无：具体执行标准/完成标志
- **示例**：
  - ✅ "帮我优化网站性能" ❌ 无性能指标
  - ✅ "处理一下这些数据" ❌ 无处理目标
  - ❌ "把这段代码格式化" ✅ 有明确操作标准

### **❌ 不触发场景（精准避让）**

| 场景类型 | 示例 | 避让原因 |
|---------|------|----------|
| **事实性问答** | "OpenClaw怎么部署Skill" | 有明确技术答案 |
| **简单指令** | "帮我整理这段文字" | 操作标准明确 |
| **用户明确要求** | "直接处理，不用确认" | 尊重用户选择 |
| **技术细节询问** | "Python列表怎么排序" | 具体问题具体答案 |
| **代码调试** | "这段代码报错怎么解决" | 有明确错误信息 |

### **规则2：边界梳理（优先级排序）**
```
核心目标（必须达成）→ 约束条件（时间/工具/权限）→ 输出要求（格式/粒度）→ 次要补充
```

### **规则3：反问规则（降低交互成本）**

#### **分步提问 - 首次仅问2-3个核心问题**
```
❌ 糟糕："你需要什么功能？用什么技术？多长时间？什么格式？预算多少？"
✅ 优秀："为了开发用户管理系统，需要确认：1）核心功能（示例：用户注册/登录/权限）2）技术栈偏好（示例：Node.js/Python）"
```

#### **引导式提问 - 用"为了精准完成"代替"你没说"**
```
❌ 糟糕："你没说用什么数据库"
✅ 优秀："为了精准完成用户管理系统，需要确认：数据库类型（示例：MySQL/MongoDB/PostgreSQL）"
```

#### **拒绝抽象 - 每个问题附带具体示例**
```
❌ 糟糕："数据范围是什么？"
✅ 优秀："数据范围（示例：近3个月销售数据/2024年全年数据/全量用户数据）"
```

### **规则4：异常应对（闭环逻辑）**

#### **用户表示"不想回答/直接做"**

## ⚙️ 技术实现要点

### **触发检测（轻量级）**
```python
def detect_fuzzy_task(user_input, context):
    # 任务类检测
    task_verbs = ["做", "开发", "分析", "设计", "规划", "构建", "实现", "优化", "处理", "完善"]
    
    # 决策类检测  
    decision_patterns = ["选哪个", "怎么选", "是否可行", "哪个更好", "哪个合适"]
    
    # 模糊类检测
    vague_patterns = ["帮我处理", "优化一下", "完善一下", "搞一下", "弄一下"]
    
    # 避让检测
    if "直接" in user_input and ("做" in user_input or "处理" in user_input):
        return False  # 用户明确要求直接处理
        
    # 综合判断
    return has_task_verb_but_unclear_scope(user_input) or \
           has_decision_without_criteria(user_input) or \
           has_vague_instruction(user_input)
```
