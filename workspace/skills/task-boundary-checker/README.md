# 复杂任务边界校验技能（精准版）

## 🎯 技能定位

**智能需求澄清助手** - 自动识别模糊需求，通过精准提问将复杂任务转化为可执行方案，避免返工和误解。

## 🚀 核心优势

| 传统方式 | 边界校验技能 |
|---------|-------------|
| 直接执行，结果偏差 | 事前澄清，精准匹配 |
| 多次返工，效率低下 | 一次确认，高效完成 |
| 用户表达不清 | 智能引导，逐步细化 |
| 需求理解错误 | 双向确认，确保一致 |

## 📋 触发场景（自动识别）

### **🎯 任务类 - 动词+模糊目标**
- **关键词**：做/开发/分析/设计/规划/构建/实现/优化
- **缺失信息**：目标/范围/验收标准
- **示例**：
  - ✅ "帮我开发用户管理系统" → ❌ 功能范围？
  - ✅ "分析一下销售数据" → ❌ 分析维度？

### **🤔 决策类 - 选择+无标准**
- **关键词**：选哪个/怎么选/是否可行/哪个更好
- **缺失信息**：比较维度/评估标准
- **示例**：
  - ✅ "选哪个框架好？" → ❌ 业务场景？
  - ✅ "这个方案可行吗？" → ❌ 可行性标准？

### **📝 模糊类 - 笼统指令**
- **关键词**：帮我处理XX/优化XX/完善XX
- **缺失信息**：执行标准/完成标志
- **示例**：
  - ✅ "优化网站性能" → ❌ 性能指标？
  - ✅ "处理这些数据" → ❌ 处理目标？

## ❌ 智能避让场景

| 场景类型 | 示例 | 处理方式 |
|---------|------|----------|
| **事实性问答** | "Python怎么排序" | 直接回答技术点 |

## 💡 智能提问策略

### **🎯 分步提问 - 降低认知负担**
```
❌ 糟糕："你需要什么功能？用什么技术？多长时间？"
✅ 优秀："为了开发用户管理系统，需要确认：1）核心功能 2）技术偏好"
```

### **🤝 引导式提问 - 避免指责**
```
❌ 糟糕："你没说用什么数据库"
✅ 优秀："为了精准完成用户管理，需要确认：数据库类型（示例：MySQL/MongoDB）"
```

### **📖 拒绝抽象 - 每个问题带示例**
```
❌ 糟糕："数据范围是什么？"
✅ 优秀："数据范围（示例：近3个月销售数据/2024年全年数据）"
```

def detect_fuzzy_task(user_input, context):
    # 任务类检测
    task_verbs = ["做", "开发", "分析", "设计", "规划", "构建", "实现", "优化", "处理", "完善"]
    
    # 决策类检测  
    decision_patterns = ["选哪个", "怎么选", "是否可行", "哪个更好", "哪个合适"]
    
    # 模糊类检测
    vague_patterns = ["帮我处理", "优化一下", "完善一下", "搞一下", "弄一下"]
    
    # 避让检测
    if "直接" in user_input and ("做" in user_input or "处理" in user_input):
        return False  # 用户明确要求直接处理
        
    # 综合判断
    return has_task_but_unclear_scope(user_input) or \
           has_decision_without_criteria(user_input) or \
           has_vague_instruction(user_input)
